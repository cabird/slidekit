*A style-agnostic checklist for AI-assisted slide review. Use alongside the presentation manifest to identify visual and structural issues.*

---

### How to Use This Checklist

When reviewing slides:
1. Read the presentation manifest first to understand the intended style, colors, and layout patterns
2. Review each slide JPEG against this checklist
3. Apply style-specific rules based on what the manifest describes (e.g., if the style uses accent bars, check their visibility)

For each issue found, provide:
- **What's wrong** (specific element, current state)
- **Why it matters** (projection visibility, hierarchy, clarity, professionalism)
- **How to fix** (specific recommendations: sizes, colors, spacing, structure)

---

### 1. OVERLAPS & COLLISIONS

- [ ] Text overlapping shapes, images, or other text
- [ ] Lines or decorative elements passing through words
- [ ] Stacked elements bleeding into each other
- [ ] Source citations or footers colliding with content above
- [ ] Decorative elements positioned for single-line text but title wrapped to multiple lines
- [ ] Icons or graphics overlapping text they shouldn't

---

### 2. TEXT & CONTENT ISSUES

#### Overflow & Boundaries
- [ ] Text cut off at edges or box boundaries
- [ ] Text boxes too narrow causing excessive line wrapping
- [ ] Content extending beyond slide edges
- [ ] Text running outside its intended container

#### Placeholder & Incomplete Content
- [ ] Leftover placeholder text (Lorem ipsum, "Title here", sample content)
- [ ] Missing images showing broken image icons or empty frames
- [ ] Incomplete sentences or obviously draft content

#### Readability at Distance
- [ ] Important text too small to read from back of room (projection test: would this be readable from 30 feet?)
- [ ] "Muted" or "secondary" text so faint it's invisible on projectors
- [ ] Footnotes or citations that are purely decorative vs. meant to be read
- [ ] Labels or captions below minimum readable threshold

---

### 3. SPACING & LAYOUT

#### Margins & Edges
- [ ] Insufficient margin from slide edges (content too close to borders)
- [ ] Elements touching or nearly touching slide boundaries

#### Element Spacing
- [ ] Elements too close together (cramped, hard to distinguish)
- [ ] Cards, boxes, or sections nearly touching each other
- [ ] Inconsistent gaps between similar elements
- [ ] Uneven distribution (large empty area in one place, cramped in another)

#### Empty Space
- [ ] Large unused areas that make the slide feel incomplete
- [ ] Content clustered in one region with empty space elsewhere
- [ ] Vertical imbalance (all content at top, empty bottom half)

#### Alignment
- [ ] Columns or rows not aligned consistently
- [ ] Similar elements at different positions or sizes
- [ ] Text baselines not aligned across columns
- [ ] Centered content that appears off-center due to optical imbalance

---

### 4. VISUAL HIERARCHY

#### Focal Point
- [ ] No clear primary element (everything competes for attention)
- [ ] Multiple elements of equal visual weight confusing the message
- [ ] Supporting elements more prominent than the main point

#### Size Hierarchy
- [ ] Headlines not significantly larger than body text
- [ ] Key numbers or stats not emphasized
- [ ] Labels larger than the content they describe

#### Emphasis Patterns
- [ ] Key phrases or takeaways not visually distinguished
- [ ] Action items blending in with explanatory text
- [ ] Important information buried in dense paragraphs

---

### 5. CONTRAST & VISIBILITY

#### Text Contrast
- [ ] Light text on light backgrounds
- [ ] Dark text on dark backgrounds
- [ ] Gray text too similar to background color
- [ ] Text on images or patterns without sufficient contrast treatment

#### Element Contrast
- [ ] Icons too similar in color to their background
- [ ] Lines or borders too thin or faint to see
- [ ] Chart elements (bars, lines, points) hard to distinguish
- [ ] Subtle differences that won't survive projector washout

#### Projection Robustness
- [ ] Colors that will wash out on typical conference projectors
- [ ] Fine details that will disappear at distance
- [ ] Thin lines (< 2-3px) that may not render clearly

---

### 6. DIAGRAMS & DATA VISUALIZATION

#### Axis & Labels
- [ ] Axis labels too long or technical (move details to notes)
- [ ] Missing directional cues (which end is "high" vs "low"?)
- [ ] Vertical text that's hard to read (prefer horizontal when possible)

#### Chart Readability
- [ ] Legend too small or hidden
- [ ] Data labels illegible
- [ ] Too many categories making it hard to parse
- [ ] Colors that are hard to distinguish from each other

#### Framework Diagrams (matrices, quadrants, flows)
- [ ] Zone or section labels hard to read against their backgrounds
- [ ] Inactive/dimmed areas too faint to understand the structure
- [ ] Borders or frames too thin to establish structure
- [ ] Overlays or annotations more prominent than the underlying diagram

---

### 7. CONSISTENCY

#### Within Slides
- [ ] Mixed font styles without clear purpose
- [ ] Inconsistent spacing patterns
- [ ] Color usage that doesn't follow established patterns

#### Across Slides
- [ ] Similar elements styled differently on different slides
- [ ] Inconsistent heading sizes or positions
- [ ] Layout patterns that shift without reason

#### With Manifest/Design System
- [ ] Elements that violate the stated design language
- [ ] Colors outside the defined palette
- [ ] Typography not matching specified fonts/weights

---

### 8. CONTENT & COMMUNICATION

#### Clarity
- [ ] Long sentences that could be split for emphasis
- [ ] Key messages buried rather than highlighted
- [ ] Technical jargon without context

#### Paired/Comparative Elements
- [ ] Comparisons without clear visual distinction
- [ ] Contrasting items not formatted symmetrically
- [ ] Missing connective elements between related pieces

#### Actionability
- [ ] Problems stated without solutions
- [ ] Frameworks without clear "so what" guidance
- [ ] Takeaways that are implicit rather than explicit

#### Evidence & Attribution
- [ ] Quotes or data without visible sources
- [ ] Citations styled as invisible fine print
- [ ] Claims that need but lack supporting reference

---

### 9. SLIDE-TYPE CONSIDERATIONS

*Apply these based on the slide's purpose:*

#### Title/Opening Slides
- [ ] All text levels readable (not just decorative)
- [ ] Clear hierarchy from title through metadata

#### Data/Statistics Slides
- [ ] Key numbers prominently displayed
- [ ] Supporting context visible but secondary
- [ ] Units and labels close to their values

#### Quote Slides
- [ ] Quote text sized to fill available space appropriately
- [ ] Attribution clearly visible as evidence
- [ ] Line length appropriate for readability

#### Diagram/Framework Slides
- [ ] Diagram occupies appropriate canvas space
- [ ] Labels integrated and legible
- [ ] Legend or key visible if needed

#### Closing/CTA Slides
- [ ] QR codes have sufficient quiet zone and contrast
- [ ] URLs large enough to photograph
- [ ] Call to action clear

---

### 10. FINAL PROJECTION TEST QUESTIONS

For each slide, ask:
1. Can I read everything from the back of a conference room?
2. Is there one clear focal point?
3. Does the visual hierarchy match content importance?
4. Is there wasted space that could be better used?
5. Would this survive a washed-out projector?
6. Are there any overlapping or colliding elements?
7. Is anything cut off, cramped, or overflowing?

---

### Style-Specific Considerations

After checking the generic items above, review the presentation manifest for style-specific rules. Look for issues related to:
- The specific color palette and its proper usage
- Typography choices and their correct application
- Layout archetypes defined in the manifest
- Any custom components or patterns described
- Icon or image styles specified

Apply the same rigor to style-specific elements as to the generic checklist items.
